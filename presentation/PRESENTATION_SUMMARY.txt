🚀 HOW TO USE
═══════════════════════════════════════════════════════════════════════════════

Step 1: READ THE MATERIALS
   → Start with README.md (master guide)
   → Review PITCH_DECK.md (main content)

   → Memorize QUICK_REFERENCE.md (key talking points)

Step 2: CUSTOMIZE FOR YOUR AUDIENCE
   → Investors: Focus on slides 1-5, 12-14, 18-21, 28-30

   → Enterprise: Emphasize slides 16, 19, 25 (use cases, ROI)
   → General: Keep it simple - slides 1-5, demo, CTA

Step 3: CREATE VISUALS (Optional but Recommended)
   → Follow VISUAL_ASSETS_CHECKLIST.md
   → Priority 1: Architecture diagram, screenshots, PDFs
   → Priority 2: Demo video, metrics dashboard
   → Priority 3: Interactive demo, branding

Step 4: CONVERT TO SLIDES
   Option A: Manual (PowerPoint, Google Slides, Keynote)
      - Copy content from markdown
      - Add visuals, format, design
      - 2-4 hours of work

   Option B: Automated (Marp, Pandoc)
      ```bash
      marp PITCH_DECK.md -o pitch-deck.pdf
      pandoc PITCH_DECK.md -o pitch-deck.pptx
      ```

   Option C: AI-Assisted (Gamma, Pitch, Beautiful.ai)
      - Paste markdown content
      - AI generates slides automatically
      - Tweak design and export

Step 5: PRACTICE THE DEMO
   → Read DEMO_SCRIPT.md thoroughly (15 min)
   → Set up equipment (Android device, sensor, mirroring)
   → Practice run 3 times (5 min each = 15 min)
   → Prepare backup video (2 min)
   Total prep time: ~45 minutes

Step 6: PREPARE BACKUP MATERIALS
   → Export ONE_PAGER.md to PDF (for handouts)
   → Generate QR codes (GitHub, APK download, contact)
   → Print QUICK_REFERENCE.md (laminate if possible)
   → Load demo video on backup USB drive

Step 7: PRESENT WITH CONFIDENCE
   → Follow timing guides (5/20/60 min versions)
   → Use power phrases from QUICK_REFERENCE.md
   → Run live demo if possible, video if not
   → End with clear call to action

Step 8: FOLLOW UP
   → Share materials (PDF one-pager, demo video)
   → Schedule 1-on-1 meetings
   → Track engagement (GitHub stars, APK downloads)
   → Send thank-you emails within 24 hours

💡 PRO TIPS
═══════════════════════════════════════════════════════════════════════════════

1. START WITH THE PROBLEM
   → "$6 trillion lost to identity fraud" grabs attention
   → Personal stories resonate more than technology

2. LEAD WITH THE DEMO
   → "Let me show you it working" beats 20 slides
   → 5-minute live demo > 30-minute explanation

3. SIMPLIFY FOR AUDIENCE
   → Investors care about market/traction, not cryptography
   → Technical audiences want deep-dives, not business models
   → Adapt on the fly based on questions

4. BACKUP EVERYTHING
   → Hardware fails, internet drops, apps crash
   → Video demo, printed slides, offline mode
   → "This is why we have 1,561 tests!" (laugh it off)

5. END WITH CLEAR CTA
   → "Can I send you the data room?"
   → "Let's schedule a pilot?"
   → "Download the app and try it?"
   → Always give them next steps

6. FOLLOW UP FAST
   → Within 24 hours, send materials
   → Within 48 hours, schedule next meeting
   → Within 1 week, close or disqualify
